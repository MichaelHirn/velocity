(

  • [options]
)

Defined in src/core.js:195

Registers a testing framework plugin via a Meteor method.

Parameters:

  • name String

    The name of the testing framework.

  • [options] Object optional

    Options for the testing framework.

    • disableAutoReset String

      Velocity's reset cycle will skip reports and logs for this framework It will be the responsibility of the framework to clean up its **!

    • regex String

      The regular expression for test files that should be assigned to the testing framework. The path relative to the tests folder is matched against it. The default is "name/.+.js$" (name is the testing framework name).

    • sampleTestGenerator Function

      sampleTestGenerator returns an array of fileObjects with the following fields:

      • path String
        relative path to place test file (from PROJECT/tests)
      • contents String
        contents of the test file the path that's returned

velocity/mirrors/request

(
  • options
)

Defined in src/mirrors/Mirror.js:40

Starts a new mirror if it has not already been started, and reuses an existing one if it is already started.

This method will update the VelocityMirrors collection with once the mirror is ready.

Parameters:

  • options Object

    Options for the mirror.

    • framework String

      The name of the calling framework

    • [testsPath] String optional

      The path to tests for this framework. For example "jasmine/server/unit". Don't include a leading or trailing slash.

    • [args] String optional

      Additional arguments that the mirror is called with It accepts all the options that are available for meteor run.

    • [env] String optional

      Additional environment variables that the mirror is called with.

    • [port] String optional

      String use a specific port

    • [rootUrlPath] String optional

      Adds this string to the end of the root url in the VelocityMirrors collection. eg. /?jasmine=true

    • [nodes] String optional

      The number of mirrors required. This is used by distributable frameworks. Default is 1

    • [handshake] String optional

      Specifies whether or not this mirror should perform a DDP handshake with the parent. Distributable frameworks can use this to get mirrors to behave like workers. The default is true

velocity/reports/submit

(
  • data
)

Defined in src/core.js:323

Record the results of an individual test run; a simple collector of test data.

The data object is stored in its entirety; any field may be passed in. The optional fields documented here are suggestions based on what the standard html-reporter supports. Whether or not a field is actually used is up to the specific test reporter that the user has installed.

Parameters:

  • data Object
    • name String

      Name of the test that was executed.

    • framework String

      Name of a testing framework. For example, 'jasmine' or 'mocha'.

    • result String

      The results of the test. Standard values are 'passed' and 'failed'. Different test reporters can support other values. For example, the aggregate tests collection uses 'pending' to indicate that results are still coming in.

    • [id] String optional

      Used to update a specific test result. If not provided, frameworks can use the velocity/reports/reset Meteor method to clear all tests.

    • [ancestors] Array optional

      The hierarchy of suites and blocks above this test. For example, ['Template', 'leaderboard', 'selected_name']

    • [timestamp] Date optional

      The time that the test started for this result.

    • [duration] Number optional

      The test duration in milliseconds.

    • [browser] String optional

      Which browser did the test run in?

    • [failureType] String optional

      For example, 'expect' or 'assert'

    • [failureMessage] String optional
    • [failureStackTrace] String optional

      The stack trace associated with the failure
